git

Système de gestion de version

Ce que cela permet de faire :

  • Historiser les changements
  • Collaborer avec d'autres développeur•euse•s
  • Gérer les conflits

Comment créer un repository ?


            git init

Les arbres de git

L'espace de travail

Le dossier où vous rangez votre code source

On peut ignorer certains fichiers avec le .gitignore

L'Index

Le choix des fichiers à historiser


            git add <nom-du-fichier-a-historiser>

Le HEAD

La validation des changements


            git commit -m "correction du bug bloquant #0012B"

L'exemple du mariage

  • L'espace de travail = Le chateau où sont les invités
  • L'index = Le photographe appelle la famille
  • Le commit = La prise de la photographie

Le repository distant

Pourquoi un repository distant ?

  • Pour sauvegarder son code
  • Pour partager son code

Comment lier son repository local au distant ?


            git remote add origin <url-de-votre-repo-distant>

Comment pousser les commits locaux au repository distant ?


            git push origin master

Les branches

Qu'est-ce qu'une branche ?

  • Espace de travail différent
  • Ayant une base de code commune

Comment créer une branche ?


            git checkout -b nouvelle-fonctionnalite-top-moumoutte

Comment revenir sur une branche existante ?


            git checkout master

Comment fusionner des changements sur une branche ?


            git merge nouvelle-fonctionnalite-top-moumoutte
